# AutoPenX 使用说明

## 简介

AutoPenX 是一个 LLM 驱动的智能渗透测试 + CTF 自动解题平台，集成 DeepSeek v4-pro 深度推理能力，支持 Web/Pwn/Crypto/Misc/Reverse 五大题型。

---

## 快速开始

### 1. 环境要求

- **操作系统**: Windows 10/11、Linux、macOS
- **Python**: 3.10 或更高版本
- **网络**: 需要访问 https://api.deepseek.com（LLM 功能）

### 2. 首次安装（新电脑）

1. 安装 Python 3.10+，**务必勾选 "Add Python to PATH"**
2. 双击 **`一键安装依赖.bat`**，等待自动完成
3. 编辑 `.env` 文件，填入你的 DeepSeek API Key：
   ```
   DEEPSEEK_API_KEY=sk-你的密钥
   ```
4. 双击 **`一键启动Web界面.bat`** 打开浏览器使用

### 3. 日常使用

- **启动**: 双击 `一键启动Web界面.bat`
- **检查环境**: 双击 `检查环境依赖.bat`
- **停止**: 关闭标题为 "AutoPenX Web" 的命令行窗口

## 功能模块

### 渗透测试模式

在 Web 界面的"启动扫描"区域：
1. 输入目标 URL（如 `http://testphp.vulnweb.com`）
2. 选择扫描模式（被动/主动）
3. 如需登录，点击 **🌐 浏览器登录** 按钮，在弹出的浏览器中手动登录
4. 点击 **▶ 开始扫描**

### CTF 解题模式

在 Web 界面的"CTF 模式"区域：
1. **上传题目文件**: 拖拽或点击上传 .py/.zip/.pcap/.png/.elf 等附件
2. **选择工具**: 勾选需要启用的外部工具
3. **配置参数**:
   - 目标地址（URL 或 `nc host port`）
   - 题目类型（自动检测/Web/Pwn/Crypto/Misc/Reverse）
   - Flag 格式（默认 `flag{...}`）
   - 最大尝试次数和超时时间
   - Thinking 模式开关（深度推理）
4. 如需登录靶场，点击 **🌐 浏览器登录**
5. 点击 **🏁 开始解题**

### 浏览器登录助手

当目标需要登录时：
1. 点击 **🌐 浏览器登录** 按钮
2. 系统弹出可见的 Chromium 浏览器窗口
3. 在浏览器中手动完成登录（支持验证码、二步验证等）
4. 登录成功后浏览器自动关闭，Cookie 被捕获
5. 后续所有请求自动携带登录凭证

## 支持的 CTF 题型

| 题型 | 内置工具 | 外部工具（可选） |
|------|---------|----------------|
| Web | SSTI/LFI/反序列化/Flag读取 | nmap, sqlmap |
| Pwn | checksec/ROP/格式化字符串/远程交互 | ROPgadget, pwntools |
| Crypto | RSA攻击/古典密码/编码解码/脚本执行 | gmpy2, z3-solver |
| Misc | 文件分析/隐写/流量/压缩包 | binwalk, steghide, tshark |
| Reverse | 反编译/字符串/动态分析/约束求解 | Ghidra, ltrace, z3 |

## 配置说明

编辑 `.env` 文件：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥 | （必填） |
| `DEEPSEEK_MODEL` | 模型名称 | deepseek-v4-pro |
| `AUTOPENX_SCAN_MODE` | 扫描模式 | active |
| `AUTOPENX_ALLOW_LOCAL_TARGETS` | 允许本地目标 | false |
| `AUTOPENX_EXPLOIT_ENABLED` | 允许漏洞利用 | false |
| `CTF_DEFAULT_TIMEOUT` | CTF 解题超时（秒） | 600 |
| `CTF_MAX_ATTEMPTS` | CTF 最大尝试次数 | 10 |

## 外部工具安装（可选）

这些工具不是必须的，缺少时系统会使用内置替代方案：

### Windows

```bash
# Python 包（在 venv 中安装）
.venv\Scripts\pip install gmpy2 pycryptodome z3-solver ROPgadget

# Nmap
# 下载: https://nmap.org/download

# Binwalk
pip install binwalk

# Wireshark (含 tshark)
# 下载: https://wireshark.org
```

### Linux (Ubuntu/Debian)

```bash
sudo apt install nmap binwalk steghide tshark exiftool ltrace strace
sudo apt install checksec john fcrackzip
pip install gmpy2 pycryptodome z3-solver ROPgadget pwntools
```

## 常见问题

### Q: 启动 bat 文件闪退？
A: 右键用"以管理员身份运行"，或在 cmd 中手动执行查看错误信息。

### Q: pip install 报编码错误？
A: 确保 `requirements.txt` 是纯 ASCII 编码（无中文注释中的特殊字符）。

### Q: Playwright 浏览器打不开？
A: 运行 `.venv\Scripts\playwright install chromium` 安装浏览器。

### Q: LLM 不工作？
A: 检查 `.env` 中的 `DEEPSEEK_API_KEY` 是否正确填写。

### Q: 端口 8000 被占用？
A: 关闭占用端口的程序，或修改 bat 文件中的端口号。

### Q: 移植到新电脑后无法运行？
A: 删除 `.venv` 文件夹，重新双击 `一键安装依赖.bat`。
