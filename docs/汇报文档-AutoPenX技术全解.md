# AutoPenX 技术全解 — 汇报文档

## 一句话介绍

AutoPenX 是一个**全自动 CTF Web 解题 AI 系统**。你给它一个网址，它自己找漏洞、构造攻击、拿到 flag，全程不需要人参与。

---

## 一、为什么做这个？

CTF（Capture The Flag）是网络安全竞赛，选手需要找到隐藏在网站中的"旗帜"（flag）。传统做法是人工一步步分析、尝试、利用漏洞。

**痛点**：
- 人工解题慢（一道题 30 分钟到几小时）
- 需要大量安全知识储备
- 重复性工作多（每次都要手动测试 SQL 注入、文件包含等）

**AutoPenX 的目标**：让 AI 像一个经验丰富的安全专家一样，全自动完成整个解题过程。

## 三、技术架构（通俗版）

想象 AutoPenX 是一个**黑客战队**，有明确的分工：

### 第一步：快速侦察（5 秒）

```
你给它一个网址
    ↓
19 个"侦察兵"同时出动（并行状态机）
    ↓
每个侦察兵负责一种攻击类型：
  - SQL 注入侦察兵
  - 文件包含侦察兵
  - 命令注入侦察兵
  - 反序列化侦察兵
  - ... 共 19 种
    ↓
4 秒后汇报："SQL 注入可能性 75%，源码泄露可能性 85%"
```

### 第二步：AI 攻击手出动（主力）

```
根据侦察结果，派出 3 个 AI 攻击手
    ↓
攻击手 A：走 SQL 注入路线
攻击手 B：走源码泄露路线（变体 1）
攻击手 C：走源码泄露路线（变体 2）
    ↓
三个人同时干活，谁先拿到 flag 谁赢
其他人立刻停手（节省 API 费用）
```

### 第三步：经验记录

```
解题成功后：
  - 记住"这种网站用 SQL 注入最快"
  - 保存成功的攻击代码模板
  - 下次遇到类似网站，直接用模板先试
```

## 四、核心创新点（6 个）

### 创新 1：确定性 + 概率性混合

**通俗解释**：先用"死规则"快速排除不可能的方向，再让 AI 在最有可能的方向上发挥创造力。

- 状态机（死规则）：零成本，4 秒扫完 19 种漏洞类型
- LLM（创造力）：只在有希望的方向上花钱调用 API

**好处**：比纯 AI 方案节省 80% 的 API 费用。

### 创新 2：并行竞速 + 第一个赢

**通俗解释**：不是一条路走到黑，而是同时走 3 条路，谁先到终点算谁的。

- 3 个 AI Worker 同时攻击不同方向
- 最有希望的方向派 2 个人（用不同方法）
- 任何一个找到 flag → 其他人立刻停

**好处**：理论加速 3 倍，实际加速 2-2.5 倍。

### 创新 3：经验沉淀正反馈

**通俗解释**：AI 会"记住"以前怎么解题的，下次遇到类似的题直接用老方法。

- 成功的攻击代码 → 保存为"快速通道"
- 失败的方向 → 降低优先级
- 目标特征 → 映射到最佳攻击路线

**好处**：第二次遇到同类题，速度提升 50-80%。

### 创新 4：1M 上下文 + 自动压缩

**通俗解释**：AI 能记住整个解题过程中的所有尝试，不会"忘记"之前试过什么。

- DeepSeek V4 Pro 支持 100 万 token 上下文
- 相当于能记住一本 500 页的书的内容
- 自动压缩机制：保留关键信息，丢弃噪音

**好处**：不会重复犯错，不会忘记之前发现的线索。

### 创新 5：Kali Linux 工具集成

**通俗解释**：AI 不仅能自己写代码攻击，还能调用专业黑客工具。

- sqlmap：自动化 SQL 注入
- nmap：端口扫描
- gobuster：目录爆破
- hydra：密码爆破
- 以及 Kali Linux 中的所有工具

**好处**：遇到需要暴力破解或复杂扫描的场景，直接用专业工具。

### 创新 6：动态能力加载

**通俗解释**：AI 不是一开始就背一本厚厚的攻击手册，而是根据题目类型只加载需要的知识。

- 检测到 SQL 注入 → 加载 SQLi 攻击知识
- 检测到反序列化 → 加载 PHP 反序列化知识
- 检测到模板注入 → 加载 SSTI 攻击知识

**好处**：减少 70% 的 prompt token 消耗，让 AI 更专注。
